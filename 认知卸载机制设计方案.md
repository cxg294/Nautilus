# 认知卸载与思维资产沉淀机制设计方案

> 日期：2026-03-28
> 状态：方案阶段，待验证和落地

---

## 背景与核心问题

当前已出现高密度信息处理过载症状：频繁上下文切换导致遗忘、工作后精神疲惫、对外物刺激反应变弱。已开通 Gemini Ultra 订阅，希望充分利用额度实现两件事：

1. **回顾我做了什么**（主要需求）
2. **发现我没意识到的模式**（次要需求）

核心设计原则：**AI替你跑，不是给你加活。** 但操作负担不是唯一约束——有效性优先。

---

## 整体架构

```
采集层（三路数据）
  ├── 截屏 + 元数据 + 键盘强度（被动，自动）
  ├── Webhook 灵感捕捉（主动，已有）
  └── 多维表格会议记录（主动，通过飞书 Bot API）
        │
        ▼
处理层（自动化管道）
  ├── 本地 OCR（macOS Vision）→ 文字日志
  ├── 三路数据合流
  └── Gemini 处理 → 生成报告
        │
        ▼
产出层
  ├── 日报（每日，基于当天原始数据）
  └── 周报（每周，基于7份日报二次提炼）
        │
        ▼
沉淀层
  ├── 原始数据 → 本地存储
  └── 成品报告 → 待定（Notion / 本地 markdown）
```

## 模块一：截屏采集

### 触发逻辑

不采用固定间隔盲拍，采用**事件驱动 + 防抖**机制：

- 监听活跃窗口变化（macOS Accessibility API）
- 窗口切换后**稳定停留 3 秒**再截屏
- 快速连切多个窗口，只在最终落脚的窗口上拍一张
- 同一窗口停留超过 5 分钟，补拍一张（防止长时间阅读/写文档丢失）

### 每条记录的数据模型

| 字段 | 来源 | 说明 |
|------|------|------|
| 截屏图片 | screencapture | 当前屏幕画面 |
| 时间戳 | 系统时间 | 精确到秒 |
| App 名称 | Accessibility API | 如"飞书""Figma""VS Code" |
| 窗口标题 | Accessibility API | 如"XX项目群""XX.fig""main.py" |
| 键盘强度 | 键盘事件计数 | 每分钟击键次数，反映工作强度 |

### 关键设计考虑

- 键盘强度**不记录内容**，只记录频次——无隐私顾虑
- 键盘强度 + 截屏配合，可区分"在干活"和"在看"
- 截屏能拍到飞书聊天窗口的完整对话（比键盘监听覆盖率更高，因为包含对方消息）

## 处理链路

### 第一步：本地 OCR

截屏图片 → macOS Vision 框架本地 OCR → 带时间戳的文字日志

输出示例：
```
14:03 [飞书 - XX项目群] 讨论XX需求的排期...
14:15 [Figma - XX页面] 编辑导航组件...
14:32 [VS Code - main.py] 修改数据处理逻辑...
```

本地处理速度快、不消耗 API 额度。几百张截图几分钟完成。

### 第二步：三路数据合流

将以下数据汇总为一份结构化输入：
- OCR 文字日志 + 键盘强度曲线
- 当日 Webhook 灵感记录
- 当日多维表格会议记录

### 第三步：Gemini 生成报告

- Gemini 只处理文字，不处理图片
- 日报：基于当天合流数据，还原"今天做了哪几件事、每件事什么状态"
- 周报：基于 7 份日报，提炼模式、趋势、被忽略的事项

### 自动化

依赖 Antigravity（本地 IDE）或 Codex（原生支持定时任务）实现自动化管道。具体实现待研究。

## 存储方案

| 数据类型 | 存储位置 | 原因 |
|----------|----------|------|
| 截屏原图 | 本地 | 量大，不适合推云端 |
| OCR 文字日志 | 本地 | 中间产物 |
| 日报/周报 | 待定（Notion 或本地） | 需要可检索 |

**排除飞书**：接口封闭、资产绑定企业非个人。

## 待验证假设

以下假设尚未验证，落地前需逐一确认：

1. **macOS Vision OCR 对截屏内容的准确度** — 特别是飞书聊天、Figma 界面等场景
2. **截屏对工作内容的实际覆盖率** — 截屏只是一帧静态画面，对长文档滚动、多 tab 切换等场景可能遗漏
3. **三路数据合流的处理复杂度** — 截屏日志（视觉转文字）、灵感碎片（文字片段）、多维表格（结构化数据）格式差异大
4. **Antigravity/Codex 的定时任务能力** — 能否稳定跑每日自动化管道

## 被否决的方案及原因

| 方案 | 否决原因 | 备注 |
|------|----------|------|
| 飞书 API 被动采集 | API 权限在企业管理员手里，个人不可控 | 实质性否决 |
| 持续录音 | 噪音比极高，8小时录音可能只有20%有价值；涉及同事隐私 | 实质性否决 |
| 键盘内容监听 | 缺少对方上下文（聊天是对话，只有自己打的字），数据不完整 | 实质性否决 |
| 固定间隔截屏 | 可能拍到切屏中间态，且同一画面重复拍摄浪费资源 | 被事件驱动方案替代 |
| Limitless (原 Rewind) | 在国内网络环境下载/使用受限无法跑通 | 2026-03-30 确认否决，全面转向完全自研 |

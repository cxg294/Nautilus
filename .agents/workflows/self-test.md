# 模块自测 SOP

每个新模块开发完成后，AI 必须执行以下标准自测流程，**不得跳过直接交付给用户**。

## 前置条件

- `pnpm dev` 正在运行
- 模块代码已保存，HMR 已生效

## 自测步骤

### 1. TypeScript 编译检查

```bash
pnpm run typecheck
```

如果有类型错误，立即修复后重新检查。

### 2. 页面加载验证

使用浏览器工具访问模块页面：

1. 导航到目标页面 URL
2. 截图检查 UI 布局是否正确
3. 验证侧边栏菜单项是否正常显示（图标 + 中文标题）
4. 验证面包屑是否正确

### 3. 控制台错误检查

捕获浏览器 console 日志，检查是否有：
- JavaScript 运行时错误
- Vue 组件渲染 warning
- 网络请求 404/500 错误
- 未捕获的 Promise rejection

### 4. 交互测试

根据模块功能进行关键路径测试：
- 点击主要按钮，验证功能是否触发
- 输入框/选择器是否可交互
- 状态变化是否正确反映到 UI

### 5. 多语言验证

如果模块有自定义文案：
- 验证中文模式下所有文案正常
- 切换到英文模式验证（如可能）

### 6. 边界情况

- 空状态提示是否正常
- 按钮禁用状态是否正确
- 极端输入（如空字符串、超大数字）是否有防护

## 自测报告

自测完成后，在向用户交付前，简要说明：
- ✅ 通过的项
- ⚠️ 已知限制（如需要真实文件上传的场景）
- ❌ 发现的问题及修复情况

## 测试素材

- 测试素材存放在 `tests/fixtures/` 目录
- 如需生成测试文件（如视频、图片），可用 Canvas/FFmpeg 脚本创建
- 素材文件应尽量小（<1MB），避免仓库膨胀
